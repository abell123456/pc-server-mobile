React VS Angular

React简介

Facebook把它简单低调地定义成一个“用来构建UI的JavaScript库”

几个核心概念:

  • 1、组件和基于组件的设计流程；
  • 2、单向数据流动；
  • 3、虚拟DOM取代物理DOM作为操作对象；
  • 4、用JSX语法取代HTML模板，在JavaScript里声明式地描述UI。

React简介

带来的好处:

  • 1、前端和后端都能够从React组件渲染页面，完全解决了SEO长期困扰JavaScript单页应用的问题；
  • 2、我们可以简单直接地写前端测试而完全忘掉DOM依赖；
  • 3、组件的封装方式和单向数据流动能够极大地简化前端架构的理解难度。

React生态圈

  • 一、多平台

    browser、React-native(Android、IOS)、Server、客户端

  • 二、数据处理

    • 1、Flux：View数据变化 -> Action -> Dispatcher -> Store -> props/state -> View；
    • 2、GraphQL:解决前后端API统一的问题，RESTFUL API痛点，前端只需要描述需要返回数据的形状；
    • 3、Immutable.js:js数据结构，提升JS操作对象的性能，大型React应用考虑使用。

React生态圈（续）

三、工具

  • 1、webpack：对于React提供对Flux、ES6、LESS的支持，打包工具；
  • 2、Babel：transform ES6 to ES5，无痛使用ES6甚至是ES7新规范；
  • 3、React-hot-Reload:不用刷新页面更新代码变化，提升开发效率；
  • 4、React Develop Tool:浏览器里查看React组件结构，方便开发查错与优化。

React缺陷

  • 1、React不是一个应用开发框架，所以你需要考虑如何处理构建一个应用程序的其它方面。比如，ng的封装好的AJAX用于$http服务，$q用于应答服务，ng-show，ng-hide，ng-class和ng-if作为模板的控制语句。弥补：react-utils
  • 2、现成组件少。弥补：react-components；
  • 3、React的信条不鼓励使用双向绑定，这也给处理表单数据和编辑表格数据带来了很多痛苦。但：开始理解Flux数据流和存储，事情就变得简单、清晰和简单)；
  • 4、相比ng（AngularUI、Restangular），可用扩展少。

Angular特点

  • 1、双向数据绑定的MVVM(Model View View-Model)框架;
  • 2、模块化、自动化；
  • 3、语义标签，依赖注入；
  • 4、DOM操作与应用逻辑解耦；
  • 5、测试与开发同等看待；
  • 6、客户端与服务端解耦，代码重用；
  • 7、声明式语法而非命令式。

Angular缺点

  • 1、基于DOM的程序执行：在Angular应用的执行时，会首先扫描所有的DOM，再通过指令进行编译;
  • 2、双向数据绑定是一把双刃剑:随着组件增加，项目越来越复杂，双向数据绑定带来性能问题。
  • 3、Angular自成一体:Angular的任何操作会引起digest cycle对注册过的监听器的遍历，以实现组件动态地同步数据。这会和其它的依赖产生兼容性问题。
  • 4、学习进阶难。使用Angular需要学习大量的概念，包括但不限于：模块、控制器、指令、作用域、模板、链式函数、过滤器、依赖注入
